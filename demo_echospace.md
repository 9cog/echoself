## How to Use EchoSpace

### 1. Access EchoSpace

1. Navigate to the Deep Tree Echo interface
2. Click on the **EchoSpace** room in the map (orange room with âš¡ icon)
3. The EchoSpace Control Panel will appear on the right side

### 2. System Overview

The control panel shows real-time system state:

- **Agents**: Count of Actual (1), Virtual (spawned), and Collective agents
- **Arenas**: Real spaces (Marduk-Space), Sandboxes, and Memory spaces
- **Relations**: Active Agent-Arena connections with autonomy levels
- **Simulations**: Currently running Virtual Marduk simulations

### 3. Running a Marduk Pipeline

#### Step 1: Define Hypotheses

1. In the "Hypotheses for Virtual Marduks" section, enter hypotheses to test
2. Click "+ Add Hypothesis" to add more (test multiple strategies in parallel)
3. Example hypotheses:
   - "Implement recursive memory optimization"
   - "Deploy adaptive attention mechanisms"
   - "Enhance cognitive synergy protocols"

#### Step 2: Execute Pipeline

1. Click **"ðŸš€ Execute Marduk Pipeline"**
2. The system will:
   - Spawn Virtual Marduks for each hypothesis
   - Run parallel simulations in the sandbox
   - Aggregate results through consensus mechanisms
   - Execute final strategy with Actual Marduk (if consensus reached)

#### Step 3: Monitor Results

The pipeline results show:

- **Virtual Marduk Simulations**: Individual simulation outcomes with confidence levels
- **Consensus State**: Vote aggregation and strategic alignment
- **Actual Marduk Execution**: Real-world action taken (if consensus achieved)

## Troubleshooting

### Common Issues

1. **Memory Service Not Initialized**

   - Ensure OpenAI API key is configured
   - Check Supabase connection
   - Initialize through Memory Library room first

2. **Consensus Not Reached**

   - Lower confidence thresholds if simulations consistently fail
   - Adjust hypothesis complexity
   - Review simulation parameters

3. **Pipeline Execution Timeout**
   - Check hypothesis complexity
   - Monitor system resources
   - Review error logs in browser console
